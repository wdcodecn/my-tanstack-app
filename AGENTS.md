## Environment Variables

No environment variables are required to run the dev server, build, or preview the generated starter.

If you add server-side secrets or runtime configuration later:

- **Server-only secrets:** Use `process.env.MY_SECRET`. These are only accessible inside `createServerFn`, `createServerOnlyFn`, API route handlers, or `.server.ts` modules. Do not prefix them with `VITE_`.
- **Client-exposed variables:** Prefix with `VITE_` (e.g., `VITE_PUBLIC_API_URL`). Vite will expose these to the browser bundle via `import.meta.env.VITE_PUBLIC_API_URL`.
- **Local development:** Create a `.env` file in the project root. It is already ignored by `.gitignore`.
- **Production:** Set environment variables on the host platform; TanStack Start's Vite build will pick them up at build time for client variables and at runtime for server variables.

## Key Architectural Decisions

1. **File-based routing.** Routes live in `src/routes/` and are auto-generated into `src/routeTree.gen.ts` by the TanStack Router Vite plugin.
2. **Root document shell.** `src/routes/__root.tsx` owns the full HTML shell (`<html>`, `<head>`, `<body>`), global meta tags, stylesheet links, and the layout (Header/Footer/Scripts).
3. **Router factory.** `src/router.tsx` exports `getRouter()` so the router instance can be created separately for client and server entry points.
4. **Tailwind v4 with CSS-first config.** Styling is configured in `src/styles.css` using `@import "tailwindcss"`, `@theme`, and CSS custom properties. There is no `tailwind.config.js`.
5. **TypeScript path aliases.** Both `#/*` and `@/*` map to `./src/*` for imports.
6. **Devtools stripped in production.** The `@tanstack/devtools-vite` plugin removes devtools imports from `__root.tsx` during `vite build`.

## Known Gotchas

- `routeTree.gen.ts` is generated; edit routes by adding/changing files in `src/routes/`, then let the dev server or `pnpm generate-routes` regenerate it.
- The README mentions a `src/routes/demo/` directory, but the generated scaffold does not include one. The starter content (Header, Footer, ThemeToggle, About page) is in `src/components/` and `src/routes/`.
- `pnpm.onlyBuiltDependencies` in `package.json` is ignored by modern pnpm; dependencies are built as needed.
- The `--tailwind` CLI flag is deprecated and ignored. To remove Tailwind later, see the README's "Removing Tailwind CSS" section.
- Devtools are visible in development but stripped automatically in production builds.

## Deployment Notes

The generated app is a full-stack TanStack Start app. Deployment target is not pre-configured. To deploy, choose an adapter and follow the TanStack Start deployment skill (`@tanstack/start-client-core#start-core/deployment`). Common targets include:

- Node.js / Docker
- Cloudflare Workers
- Vercel
- Netlify
- Railway / Render
- Bun

For static/SPA deployment, configure the `ssr` option per route or globally in `src/router.tsx` / `vite.config.ts`.
